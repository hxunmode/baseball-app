# Catch to Win — 모바일 이식 구현 계획

원본: 웹 프로토타입(`index.html`) — 캔버스 기반 미니 게임 + 단순 온보딩 플로우.

현재 앱 버전은 로그인 단계를 제거하고 `닉네임 입력 -> 응원 팀 선택 -> 랜딩 -> 게임/경기 일정` 흐름으로 단순화했다.

## 1. 원본 동작 요약

| 단계 | 화면 | 동작 |
|------|------|------|
| 1 | 닉네임 | 입력 없으면 `"플레이어"` |
| 2 | 응원 팀 선택 | `assets/team` 이미지 10개를 5x2로 배치, 탭 시 랜딩으로 이동 |
| 3 | 랜딩 | `게임 시작하기`, `경기 일정보러가기` 버튼 제공 |
| 4 | 경기 일정 | 응원 팀에 맞는 팀별 달력 이미지 표시 |
| 5 | 게임 | 400×600 논리 좌표, 공 낙하·글러브 좌우 이동, 45초 타이머 |
| 6 | 리워드 성공 | 20점 이상이면 즉시 리워드 화면 + 홈으로 |
| 7 | 재도전 실패 | 20점 미만이면 점수 안내 + 다시 도전하기 |

### 게임 규칙 (수치 동일 유지)

- 공 시작 위치: `ballX = random(0, 360)`, `ballY = 0` (공 너비 40 기준).
- 매 프레임: `ballY += 5 + (45 - time) * 0.35` (`time`은 남은 초). 시간이 지날수록 더 빠르게 가속.
- 글러브: 가로 `gloveX`, 세로 `530`, 크기 `100×60`.
- **캐치**: `ballY > 520` 이고 `ballX > gloveX` 이고 `ballX < gloveX + 100` → 점수 +1, 공 리셋.
- **미스**: `ballY > 600` → 공만 리셋 (점수 변화 없음).
- 입력: 웹은 캔버스 `mousemove`에 `gloveX = clientX - rect.left - 40` — 모바일에서는 터치 X를 논리 좌표로 변환 후 동일 오프셋 적용.

## 2. 기술 선택 (Expo / React Native)

| 항목 | 선택 | 이유 |
|------|------|------|
| 렌더링 | `View` + 스타일 (캔버스 미사용) | 의존성 최소, 디버깅 쉬움, 웹과 동일한 히트박스를 그대로 표현 가능 |
| 게임 루프 | `requestAnimationFrame` + `useReducer` 강제 리렌더 | 별도 게임 엔진 없이 2D 스프라이트 수준으로 충분 |
| 타이머 | `setInterval` 1초 | 원본과 동일 |
| 좌표계 | 논리 400×600, 화면 너비에 비례 스케일 | 기기별 너비 대응, 터치 역변환으로 동일 판정 |

## 3. 디렉터리 구조 (Phase 1 완료 목표)

```
docs/
  GAME_IMPLEMENTATION_PLAN.md   ← 본 문서
src/
  game/
    constants.ts                ← 논리 크기, 색상
    GameBoard.tsx               ← 인게임 UI + rAF 루프 + 터치
App.tsx                         ← 화면 상태 머신 + 온보딩 UI
```

의존성: `react-native-safe-area-context` (온보딩·결과 화면 안전 영역).
